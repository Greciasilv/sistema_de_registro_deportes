# sistema_de_registro_deportes
sistema de registro automatizado con gestión de visualización e inscripción en torneos deportivos en Caracas (como el Torneo Guacamaya UMA y el Padel Fest UMA), resolviendo la transición de estructuras planas a un modelo robusto de Programación Orientada a Objetos (POO). El software aplica los principios de modularización al dividir el código en archivos independientes con responsabilidades únicas (usuario.py, torneo.py, gestor.py y main.py), implementa una relación de agregación donde la clase Torneo administra colecciones dinámicas de la clase Usuario, y protege sus atributos mediante encapsulamiento (_inscritos). Asimismo, optimiza las consultas a través de algoritmos de ordenamiento (Bubble Sort por precio) y búsqueda binaria recursiva por ID, garantizando la persistencia local de los datos al serializarlos en un archivo torneos.json y blindando la interfaz de usuario basada en consola (CLI) con un manejo de excepciones (try-except ValueError) a prueba de fallos.
